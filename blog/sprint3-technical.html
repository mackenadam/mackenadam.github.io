JavaScript - Functions, loops, arrays, objects, and the DOM

Sprint Three - Technical Blog Three

17 September 2020

Building a house

You can think of making a website with HTML, CSS, and JavaScript as building a house. HTML would be like the anchitect's first draft and structural engineer's report that determine the general features of the house. How many bedrooms do we have? Is there one or two toilets? Do we have a garage or do we have to deal with on-street parking? Is there more than one storey? Do we have doors and windows to the outside world or are we stuck in the same house forever? CSS on the other hand, is the interior designer providing the details. It can do everything right up from determining the size of the rooms, right down to the colour of the paint and the font of the "live, laugh, love" decal on our feature wall. JavaScript provides the utilities. Want running water, flushing toilets, and electricity supplied to your house? That's JavaScript. Normally hidden from view, it is responsible for making sure something actually happens when you flick a light switch, or turn on a tap. Although we can build our house without some of these features it's not until all of these things come together that we have a sturdy, beautiful, and functioning house.

Control flow

One of the ways JavaScript helps us to provide funtionality to our websites is through control flow and loops. How would we go about adding lights to our house? There are a hundred different ways we could achieve this, one way would be with a simple 'if' statement. In psuedo-code this might look like:

-> If the light switch is turned off
---> Turn the switch on
---> Provide electricity to the lightbulb
-> Or else if the switch is on
---> Turn the switch off
---> Make sure the lightbulb doesn't get electricity

And in vanilla JavaScript:

if (lightSwitchOn === false) {
lightSwitchOn = true;
provideElectricity = true;
} else if (lightSwitchOn === true) {
lightSwitchOn = false;
provideElectricity = false;
}

This is a very simplistic example and in a real example you would probably want to have access to a bit more information. For instance, was the light switch already switched on? You might want to make sure that the power is set to be off by default before you start playing around with switches. You might want to make it so that you could reuse these instructions for every switch in the house. We could do this by putting our if statement inside a function. In psuedo-code:

-> Set the light switch to be off to begin with
-> Make sure the light bulb isnt getting electricity
-> Make our if statement into a function that we can reuse for all light switches

As code:

var lightSwitchOn = false;
var provideElectricity = false;
function toggleLightSwitch () {
if (lightSwitchOn === false) {
lightSwitchOn = true;
provideElectricity = true;
} else if (lightSwitchOn === true) {
lightSwitchOn = false;
provideElectricity = false;
} }

Now our code tracks two global variables; one which tracks whether the light switch is on or not, and another which tracks whether electricity should be supplied to the lightbulb. Initially the values are both set to false, meaning that our light switch is off and no electricity is supplied. When we call our function if the switch is already off then it is set to on and electricity is supplied. If it is already on then the opposite is true. Furthermore, our if statement is now contained within a function that we can use for each switch in our house.

Where does the DOM come in?

The DOM, or Document Object Model, provides an interface for HTML, CSS, and JavaScript to work in unison. It provides a representation of our website comprised of nodes similar to a family tree. Starting with the Document, the term used for our HTML file, each element is displayed as a node connected to any parent or children nodes. For instance, the HEAD and BODY elements are both children of the DOCUMENT node. From here each element that is a child of the BODY is also a node, and each of their children are nodes. What this allows us to do is to target specific elements using JavaScript and use this to apply some sort of functionality or change. Returning to out light switch analogy, lets say we wanted to apply our function to the BUTTON element of the first div; the only light switch in the master bedroom. We could add an event listener that would look like:

document.getElementByID("switch1").addEventListener("click", toggleLightSwitch);

Now an event listener is set to watch our button switch1 and any time it detects a click it will run our function toggleLightSwitch. Pretty nifty!

Arrays != Objects

Often when using the DOM we will return an Array or an Object. Although they look similar there are some major differences between the two. Both store a collection of data rather than one value and both are mutable so the original data inside them can be changed. You can think of an Object as a thing in your code; like a user with values relating to a username, password, and email address. An Array is more of a catalog or list of data. Just to make things really confusing an array can store a collection of Objects (like users), and an Object can have an array as a property (like multiple email addresses).
