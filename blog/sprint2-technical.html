Core CSS - Padding, border, and margins

Sprint 2 Technical Blog 2

10 November 2010

When you first jump into CSS you might be a bit confused when it comes to padding and margins. On the surface of it they seem to both deal with the same thing; the space around the element you are trying to style. However, there are a couple of key differences between the two which make them behave very differently. In this technical blog we will look at some of the differences between the two and a few use cases.

Let's talk about borders

To describe padding and margins it is first important to understand borders. Essentially, borders are the edge of a given element. Sometimes borders are a simple 1px wide black rectangle around an element. Sometimes they are a dotted line with rounded Borders. Often the border isn't styled at all and it's impossible to see where it is without the use of developer tools. Borders take many different forms and sometimes it can be hard for a beginner to find where the border of an element lies.

Padding sounds... Soft

Now that we know what a border is we can talk about padding. Applying padding to an element gives the content inside it a 'cushion' of dead space between it and the border. Think of your content as an important object you are sending out for delivery. The padding is like the bubble-wrap surrounding your important object, and the border is the box it is delivered in. It provides a gap between the object and the box so it the package is dropped then the contents aren't damaged.
